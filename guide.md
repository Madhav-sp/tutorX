# Project Guide: TutorX AI Learning Platform

## 1. Project Overview
**TutorX** is a comprehensive, multifaceted AI-powered learning platform. Its main purpose is to empower users to learn new skills, process complex information, and practice coding through a suite of AI-driven tools. It acts as a personal tutor, course generator, and coding evaluator all in one.

## 2. Tech Stack
- **Frontend**: [Next.js](https://nextjs.org/) (App Router), [React](https://react.dev/), [Tailwind CSS](https://tailwindcss.com/)
- **Backend**: Next.js API Routes (Serverless Functions)
- **Database**: [MongoDB](https://www.mongodb.com/) with [Mongoose](https://mongoosejs.com/) (Schemas for Courses, Notebooks, Sessions, Goals, etc.)
- **Authentication**: [Clerk](https://clerk.com/) for secure user management
- **AI Engine**: [Google Gemini AI](https://ai.google.dev/) (Pro & Flash models via `@google/generative-ai`)
- **Code Execution**: [Judge0](https://judge0.com/) API for remote code compilation and evaluation
- **Key Libraries**: `@monaco-editor/react` (code editor), `lucide-react` (icons), `pdfjs-dist` (PDF parsing), `react-markdown`

## 3. Folder Structure
- `/app`: The core of the Next.js App Router.
  - `/(auth)`: Clerk authentication pages (Sign In / Sign Up).
  - `/api`: The backend. Contains modular endpoints for AI generation (`/api/generate-course`, `/api/ai`), Notebook/PDF processing (`/api/notebook`, `/api/pdf`), and user progress (`/api/progress`).
  - `/dashboard`: The main user interface post-login, presenting access to features.
  - `/course/[id]`: dynamic routes for navigating generated course content.
- `/models`: Mongoose database schemas defining the data structure (`Course`, `AIPracticeSession`, `Notebook`, `UserProgress`, `Goal`, `Roadmap`).
- `/components`: Reusable React components, organized by feature (e.g., `/components/ai-practice`) and atomic UI (`/components/ui` for Shadcn-style components).
- `/services`: Abstraction layer for complex third-party API interactions (e.g., `aiService.js` for Gemini prompt management, `judgeService.js` for Judge0 execution).
- `/lib`: Shared utilities, such as database connection logic (`db.js`) and AI client initialization (`gemini.js`).

## 4. System Architecture
- **Frontend Flow**: Client-side React components manage UI state and collect user inputs (e.g., PDF uploads, course topic prompts, code submissions).
- **Backend Flow**: Next.js API routes act as intermediaries. They verify Clerk authentication, validate inputs, interact with the MongoDB database using Mongoose, and make server-to-server calls to third-party APIs (Gemini, Judge0).
- **AI Integration**: The backend constructs complex, structured prompts based on user input and sends them to the Gemini API, forcing JSON output for predictable rendering in the frontend.
- **Database Interaction**: Data is persistently stored in MongoDB. For instance, when a course is generated by AI, the resulting JSON is parsed and saved as a `Course` document, allowing the user to return to it anytime.

## 5. Core Features
1. **AI Course Generator**: Users input a topic, and Gemini generates a full course with structured chapters, topics, reading content, code snippets, flashcards, and quizzes. 
2. **Notebooks (NotebookLLM Clone)**: Users can upload PDFs or paste text. The AI parses the text, generates a summary, creates flashcards, and provides a chat interface allowing the user to "talk" to their document and ask questions about specific sections.
3. **AI Practice Lab (DSA)**: Generates LeetCode-style coding problems based on patterns (e.g., "Two Pointers"). Features an integrated Monaco editor supporting Java, Python, and JavaScript, with real-time code evaluation against hidden test cases via Judge0.
4. **YouTube Summarizer**: Takes a YouTube video URL, fetches the transcript, and uses Gemini to generate a concise summary and learning notes.
5. **Progress & Goals**: Users can define learning goals, follow roadmaps, and the system tracks their completion percentage across courses and chapters.

## 6. Important Components / Modules
- `models/Course.js`: A massive schema defining how AI-generated educational content is structured, nested from Courses -> Chapters -> Topics -> ContentBlocks/Flashcards/Quizzes.
- `services/aiService.js`: Houses the prompt engineering logic necessary to force the Gemini AI to output strict JSON arrays, essential for parsing problems and courses into the database.
- `services/judgeService.js`: The code execution engine. It accepts raw code, dynamically wraps it in robust, language-specific runner templates (especially for Java `Main` classes and input parsing), and securely executes it via Judge0.

## 7. Data Flow (Example: Course Generation)
1. **Request**: User types "React Basics" and hits generate on the frontend.
2. **API Layer**: `/api/generate-course` receives the request.
3. **AI Processing**: The backend sends a structured prompt to Gemini, asking for a syllabus and content formatted as JSON.
4. **Database Storage**: The backend receives the JSON, parses it, creates a new `Course` document in MongoDB via Mongoose, and links it to the user's Clerk ID.
5. **Response**: The frontend receives the Course ID, redirects the user to `/course/[id]`, and fetches the newly saved course from the database to render the UI.

## 8. Setup & Running the Project
1. **Clone the repo** and install dependencies:
   ```bash
   npm install
   ```
2. **Configure Environment variables**: Create a `.env.local` file at the root:
   ```env
   MONGODB_URI=your_mongodb_connection_string
   NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY=your_clerk_pub_key
   CLERK_SECRET_KEY=your_clerk_secret
   GEMINI_API_KEY=your_google_gemini_key
   JUDGE0_URL=https://ce.judge0.com  # Or RapidAPI Judge0 URL
   # Optional: RapidAPI Key if using Judge0 via RapidAPI
   NEXT_PUBLIC_RAPIDAPI_KEY=your_key
   ```
3. **Run the development server**:
   ```bash
   npm run dev
   ```
4. **View Application**: Open your browser and navigate to `http://localhost:3000`.

## 9. Key Dependencies
- `@clerk/nextjs`: Handles complex User Authentication flows effortlessly.
- `@google/generative-ai`: The brain behind content, problem, and summary generation.
- `mongoose`: Streamlines MongoDB document interaction and schema validation.
- `@monaco-editor/react`: Provides the VS-Code like editing experience in the Practice Lab.
- `pdfjs-dist`: Used to extract text from uploaded PDF files for the Notebook feature.

## 10. Future Improvements
- **Scalability**: Implement Redis caching for frequently accessed AI-generated courses to save on Gemini API calls.
- **Microservices**: Split the heavy PDF parsing/AI generation tasks into separate background workers (e.g., using BullMQ) to prevent API route timeouts on edge functions.
- **Social Features**: Allow users to publish their generated courses or notebooks to a public "Community Hub", complete with upvoting and forking capabilities.
